【项目概述】
请帮我开发一款纯原生的 Android 应用程序。这是一款面向科研人员和 AI 开发者的“前沿论文快筛与订阅”工具。应用核心聚焦于 arXiv 论文的追踪，主打“可定制时间窗”的阅读体验，并深度集成 AI 提炼与 Zotero 工作流。

【核心逻辑变更】
不同于传统的每日推送，本应用支持用户设置“回溯天数”（例如最近 3 天、5 天或 7 天）。主页的信息流和 AI 导读将基于这个动态的时间窗口进行聚合。

【技术栈要求】

语言： Kotlin (最新版)

UI： Jetpack Compose + Material Design 3

架构： MVVM + Repository Pattern

本地存储： Room (收藏/缓存) + DataStore (用户偏好)

网络： Retrofit2 + Kotlin Coroutines & Flow

【UI 架构与功能模块】

Tab 1: 首页 (Home & Discover)

顶部交互区：

搜索栏： 支持关键词检索。

筛选 Chip 组： 第一部分是领域标签（如 cs.CV）；第二部分是一个时间窗口选择器（如：24h, 3 Days, 7 Days），切换后即刻刷新下方列表。

动态 AI 摘要卡片 (AI Trends Summary)：

位于列表最上方，采用悬浮或置顶的卡片设计。

功能： 根据选定的时间窗口（如最近 3 天），调用一次 API 生成该周期内的“学术简报”（包含最热门的 3 个研究趋势及代表作）。

交互： 支持点击展开全文，支持手动关闭（Dismiss）。

论文流 (Paper Feed)：

展示所选周期内的论文卡片。

卡片内容： 标题、作者、发布日期、所属分类。

操作按钮：

翻译：按需翻译当前摘要。

收藏：存入本地数据库。

同步：一键发送至 Zotero 远程库。

Tab 2: 收藏库 (Library)

持久化列表： 展示所有已收藏论文，支持离线查看。

状态标记： 视觉化展示是否已同步至 Zotero。

批量管理： 支持左滑删除或批量导出 BibTeX 引用。

Tab 3: 设置页 (Settings)

偏好配置： * 默认关注领域勾选。

默认时间窗口设置（例如：每次打开 App 默认看最近 3 天的论文）。

集成配置： * Zotero API Key 及 User ID 配置。

自定义大模型 API（Base URL / API Key）用于摘要总结和翻译。

外观： 完善的明亮/暗黑模式切换。

个人信息：昵称、头像等
