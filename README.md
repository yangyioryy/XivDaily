# XivDaily

XivDaily 是一个面向科研人员和 AI 开发者的论文快筛工具。
首版采用原生 Android 客户端加 Python 后端的双端结构，围绕
arXiv 论文追踪、AI 导读、收藏管理和 Zotero 集成展开。

## 当前仓库阶段

当前仓库仍处于初始化搭建阶段，优先目标是建立可并行开发的基础结构，
再逐步补齐接口协议、后端能力、Android 客户端页面和测试交付闭环。

## 目录说明

- `android/`：Android 客户端工程与 UI、数据层代码。
- `backend/`：Python 后端工程、配置、迁移与测试代码。
- `docs/`：架构、接口、运行说明和回归文档。
- `issues/`：唯一状态源 CSV，驱动逐条 issue 闭环执行。
- `plan/`：阶段计划和 MVP 范围冻结文档。
- `scripts/`：联调、检查、构建等辅助脚本。

## 开发约束

- 功能边界以 `issues/2026-04-28_21-26-06-xivdaily-fullstack-plan.csv`
  和 `plan/2026-04-28_21-26-06-xivdaily-fullstack-plan.md` 为准。
- 涉及关键流程、复杂状态转换、边界过滤、环境隔离的代码，必须补
  简体中文注释，样板代码不强行堆注释。
- 敏感信息只能从环境变量读取，项目根目录中的本机密钥文件不能提交。

## 本地开发约定

- 后端开发环境使用 `xivdaily` conda 环境。
- Android 端当前仅冻结目录职责，后续再落正式 Gradle 工程骨架。
- 后端当前仅冻结目录职责，后续再落 FastAPI 工程骨架和依赖。
