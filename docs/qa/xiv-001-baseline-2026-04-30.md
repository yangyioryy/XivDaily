## 2. `/papers` 与 `/summaries/trends` 分层归因

### `/papers` 链路

- 后端入口：`backend/app/api/papers.py`
- 当前接口：`GET /papers`
- 当前约束：`days` 为必填查询参数，范围 `1..30`，默认 `7`
- 测试证据：`backend/tests/test_papers_api.py` 覆盖分页响应，断言 `status == ok` 且 `warning is None`
- 基线结论：论文列表接口本身可以返回成功响应；后续搜索误杀主要来自 `days <= 30` 和服务层时间窗过滤。

### `/summaries/trends` 链路

- 后端入口：`backend/app/api/ai.py`
- 当前接口：`GET /summaries/trends`
- 后端降级：`backend/app/services/ai_service.py` 在 LLM 失败时返回 `status=degraded` 和本地趋势项
- 前端污染点：`HomeViewModel.refreshTrendSummary()` 在趋势请求失败时调用 `setError(...)`
- UI 影响：`HomeScreen` 使用同一个 `errorMessage` 展示全局错误卡片，导致趋势摘要失败容易和论文列表状态混在一起。
- 基线结论：`/papers` 成功和 `/summaries/trends` 失败是两条链路；当前前端状态模型没有完全隔离这两类错误。

## 4. 三页 UI 差异清单

### 首页

- 顶部区：当前存在较重的探索控制卡片和说明文字；参考图更接近标准顶部栏 + 轻量搜索框。
- 搜索区：当前 `ExploreControlCard` 外层白卡较厚；参考图要求浅灰搜索框直接落在页面背景上。
- 趋势卡：当前趋势卡与全局错误提示耦合；参考图是独立浅蓝紫趋势卡，失败时也应局部降级。
- 论文卡片：当前摘要和教学文案信息密度偏高；参考图更强调标题、作者、日期分类和黄色收藏 Icon。

### 收藏库

- 顶部区：当前有 hero 信息区；参考图要求标准标题栏、搜索 Icon 和选择入口。
- 筛选区：当前筛选和统计层级偏重；参考图要求“全部 / 已同步 / 未同步”三段状态入口并带数量。
- 卡片：当前摘要默认展示较长；参考图要求默认隐藏长摘要，右上角突出黄色星标。
- 批量区：当前批量操作不够接近固定底部栏；参考图要求选择态下底部显示全选、导出 BibTeX、删除。

### 设置页

- 顶部区：当前标题和说明偏重；参考图要求标准“设置”顶部栏。
- 资料卡：当前头像是预设选项；参考图要求可展示真实用户头像、编辑入口和右箭头。
- 配置区：当前偏状态查看；参考图要求 Zotero 和大模型配置可点入填写、保存、测试。
- 退出登录：当前存在 `退出登录（即将开放）`；首版无登录系统，后续应完全删除该入口。
