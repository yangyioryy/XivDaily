# 接口演进规则

## 目标

在 `XIV-003` 之后，所有接口和模型变更都必须遵循统一节奏，避免 Android、
后端和测试文档不同步。

## 规则

1. 先改 `docs/openapi-draft.yaml`，再改 DTO/Room/后端实现。
2. 枚举值新增或修改时，必须同步更新映射文档和回归清单。
3. 删除字段属于破坏性变更，首版开发阶段默认禁止。
4. 新接口必须标明所属页面、失败语义和是否需要本地缓存。
5. 涉及 AI 与 Zotero 的接口，必须声明超时、降级或失败信息字段。

## 回归重点

- DTO 命名是否仍与 OpenAPI 一致。
- Room 枚举值是否仍与后端一致。
- 设置页启动配置是否仍能覆盖默认分类、时间窗口和主题。
