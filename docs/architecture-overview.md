# XivDaily 架构概览

## 系统目标

XivDaily 首版聚焦一个最小闭环：围绕可配置时间窗口的 arXiv 论文浏览，
提供 AI 趋势摘要、单篇摘要翻译、本地收藏与 Zotero 同步能力。

## 双端职责

- Android 客户端负责界面、交互、本地收藏、用户偏好和后端接口消费。
- Python 后端负责论文聚合、时间窗口过滤、AI 编排、Zotero 转发和轻量持久化。

## 数据职责边界

- Room 只承载客户端离线收藏和本地缓存。
- SQLite 只承载服务端配置、同步记录和任务状态。
- 首版不做账号系统和跨设备同步，因此两端数据责任必须保持清晰。

## 协作原则

- 新需求先回写 issue CSV，再进入实现。
- Android 和后端都先搭最小可运行骨架，再逐步补功能。
- 关键流程代码要补中文注释，方便后续联调和维护。
